Github basics

형상관리? 소프트웨어 구성요소 일관성 유지하고 추적, 관리하는 것.

목표 4가지는 버전 관리, 변경 추적, 협업 지원(동시 작업할 때 충돌 방지), 품질 보장.

**Git이랑 Github 차이**

- **Git(로컬):** 분산 버전 관리 시스템. 변경 사항 기록하고 브랜치(메인, 사이드-나뭇가지) 지원함.
- **Github:** 클라우드 시스템. 원격 저장소, 코드리뷰, 이슈 트래커, 풀 리퀘스트(PR) 지원.
- Git에서 Issue는 협업 과정에서 내 업무나 처한 상황 이야기해 주는 게시판임 (기능이나 마크업 등 카테고리).
- GithubDesktop,visual studio code gui를 사용해서 나중에는 딸깍으로 쉽게
    
    (소스코드 공유 폴더에 올리고 PR 쪽에 보면 알아서 변경 내용 빨간색, 녹색으로 뜸. 데스크탑 아니면 캡쳐해서 가시성 높게 올리고, 나중에 형상관리자가 merge 하거나 담당자가 합쳐줌.)
    
    깃 기본 흐름 : working repository → stagging area → local repository
    

**Git Bash 실습 & vscode 기초**

- `$ git config --global user.name "ㅁㄴㅇ"` (하이픈 2개는 문자열 옵션, `h`처럼 1개는 1글자 옵션)
- `$ git config --global user.email "ㅁㄴㅇ.com"` (로그인 이메일이랑 동일)
- `$ git config --list` (디폴트 설정 내역 보기)
- C드라이브에 폴더 만들고 `open git bash` 열어서 `code .` 치면 vscode가 해당 경로로 열림.
    
    (숨김파일 체크하면 `.git` 폴더 보임. vscode 저장은 Ctrl+S, 이름 수정 F2, 삭제는 delete)
    

**버전 등록 (Commit)**

버전으로 등록, 즉 커밋을 해야 추적이 됨.

- `$ git add .` (전체 다 STAGGING AREA로 올리는 행위)
- `$ git commit -m 'add all'` (로컬 래포지토리로 메시지랑 같이 올릴 때. 홑따옴표 안에 이름)
- `$ git commit -am '메시지'` (추가 생성된 파일 없이 내용만 수정됐을 때, 메시지 보내면서 한큐에 커밋까지 하고 싶을 때. 자주 쓰진 않지만 쓰면 편함)

**원격 저장소 연동 & 올리기**

- Github 들어가서 repository 새로 만들고 public으로 올림.
- 로컬이랑 원격 연동은 remote 써서 함.
- `$ git push -u origin main` (처음에 어떤 원격지, 어떤 브랜치로 올릴지 타겟 지정해 주는 거. 한번 해두면 나중엔 푸쉬만 해도 알아서 올라감)

**버전 되돌리기**

- `$ git reset --hard ~~` (비추하지만 알려주겠다. 원하는 되돌리기 시점의 커밋 버전으로 싹 돌림)

**원격지 파일 수정 & 가져오기 (Pull)**

- 깃허브 웹에서 파일 들어가서 `Edit this file` 누르면 원격으로 바로 수정 가능. `Add file` 눌러서 파일 추가도 됨.
- 이렇게 원격지(Github)에서 수정된 내용이 생겼다면, 로컬 깃 배쉬에서 `git merge`로 내려받으면 됨.
- 근데 패치(Fetch)랑 머지(Merge)를 한 번에 동시에 하려면? 그냥 `$ git pull` 쓰면 됨.

**코드 충돌 (Conflict) 상황 & 해결법**

- 충돌은 같은 파일을 수정했을때 났을 확률이 높다.!
- **해결 과정:**
    1. 일단 내가 로컬에서 수정한 걸 staging area에 올리고(`add`), `commit` 한 다음 `push`까지 해보자.
    2. 그럼 에러가 뜨면서 힌트를 줌.    `pull` 먼저 하고 `push` 해라"라고 알려준다 }
    3. 그래서 `$ git pull`을 땡겨보면 에디터(VS Code 등)에서 충돌 났다고 4가지 옵션을 추천해 줌. (내 거 쓸래? 들어오는 거 쓸래? 둘 다 합칠래? 등등)
    4. 상황 보고 판단해서 선택한 다음, 변경된 내용 바탕으로 `$ git commit -am "메시지"` (add랑 commit 한큐에 해결) 하고 다시 `push` 하면 충돌 마무리됨.

**로컬 환경이 바뀌었을 때 (Clone 실습)**

- 자리나 컴퓨터가 바뀌었을 때 깃허브에서 `<> Code` 버튼 누르고 `~~~.git` URL 복사해서 가져오면 됨.
- `$ git clone [복사한 URL]`
- 만약 불러올 때 로컬 폴더 이름까지 내 맘대로 조금 수정하고 싶다면,  띄어쓰기하고 이름 적어주면 됨.
    
    (`$ git clone [복사한 URL] [새로운폴더명]`)
    

**리눅스 명령어 기초 (cd)**

- 클론하고 나면 엉뚱한 데 있을 수 있으니까 깃 배쉬에서 `cd`(Change Directory) 써서 해당 폴더로 진입해 줘야 된다.
- `$ cd [폴더명]` (예: `$ cd ./git-test`): 내가 원하는 경로로 더 들어가기.
- `$ cd ..` : 이건 실행 취소가 아니라, 현재 폴더에서 상위 폴더(뒤)로 빠져나오는 명령어.

## 자주 사용하는 명령어

- 사용자 정보 등록하기 (Git 세팅할 때 최초에만)
git config --global [user.name](http://user.name/) "사용자이름"
git config --global user.email "사용자이메일"
git config --list
- 로컬저장소 만들기 (깃 작업 대상으로 만들기, 최초에만)
git init
- 특정 원격저장소와 연결하기(연동 시 최초에만)
git remote add origin <원격지주소>
- 작업 파일 원격지에 올리기
git add <파일명> .
git commit -m "커밋메시지"
git push [원격지명 원격브랜치명]
- 원격지에 있는 것 내려받기
git pull
